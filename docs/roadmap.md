# Slabs Roadmap

## Phase 0: Foundation

- Establish the core vocabulary.
- Publish the first public repo.
- Create starter templates.
- Seed the initial bootstrap project.

## Phase 1: Operating Discipline

- Tighten slab quality criteria.
- Define status conventions more formally.
- Add examples for different kinds of work.
- Clarify how memory should be captured and referenced.

## Phase 2: Automation Layer

- Draft automation prompts for proactive micro slab suggestions.
- Define when suggestions should be generated.
- Define how suggestions are reviewed, accepted, rejected, or refined.
- Connect automation behavior to recent tracking and memory.

## Phase 3: Lightweight Tooling

- Generate slab files from templates.
- Create indexes or rollups from Markdown sources.
- Add validation for links and identifiers.
- Explore metadata formats that remain human-readable.

## Phase 4: Workflow Packs

- Product and engineering pack
- Research pack
- Writing pack
- Operations pack
- Personal planning pack
