# Macro Slab: M04 Design The Proactive Suggestion Loop

## Status

`Ready`

## Parent Brief

- [`../work-brief.md`](../work-brief.md)

## Objective

Define how Codex automations and proactive workflows should generate suggested micro slabs during off-hours without taking decision-making away from the user.

## Why It Matters

This is the mechanism that turns Slabs from a passive documentation structure into a momentum-building operating system.

## In Scope

- suggestion inputs
- candidate micro slab output format
- review and acceptance flow
- role of memory and tracking in automation prompts

## Out of Scope

- full autonomous execution
- production scheduler implementation

## Dependencies

- stable enough brief, tracking, and memory formats
