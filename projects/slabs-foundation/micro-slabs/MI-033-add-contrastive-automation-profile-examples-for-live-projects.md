## Goal

Add compact example automation profiles that show how a docs-first project and
an app-first project should differ when declaring primary repositories, skills,
plugins, and context notes for hourly review.

## Working Context

The repo now has two grounded examples in `slabs-foundation` and
`roundreserve`, and template guidance alone may still leave future authors
guessing which fields really matter. A contrastive example can show why one
project only needs the Slabs repo while the other points into
`LovableRoundReserve` and operational skills.

## Done Criteria

- repo docs include concise example automation-profile excerpts for both live
  projects
- the examples explain why the repo, skill, and plugin choices differ across
  project types
- the examples link back to the main template rather than becoming a competing
  pattern
- the examples stay grounded in the current live project contracts
