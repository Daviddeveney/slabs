# AGENTS.md

## Intent

Slabs is a Markdown-native framework for structuring Codex work. When working in this repo, optimize for clarity, small task boundaries, durable handoffs, and lightweight progress updates that reduce context switching.

## Working Rules

1. Treat the active work brief as the source of truth for the initiative.
2. Keep macro slabs outcome-oriented and micro slabs execution-oriented.
3. Keep micro slabs small enough to finish in one focused session whenever practical.
4. Update the tracking hub when a micro slab changes state or produces a meaningful artifact.
5. Capture durable context in Markdown memory files instead of relying on chat history.
6. Preserve history. Mark stale content as superseded instead of deleting useful context.

## Editing Guidance

- Prefer evolving existing templates and docs over creating parallel patterns.
- Link related slabs together explicitly.
- Record decisions where future agents can find them quickly.
- Keep examples realistic enough that a cloned repo can be used immediately.

## Status Language

Use one of these states unless the surrounding document defines a tighter set:

- `Backlog`
- `Ready`
- `Active`
- `Blocked`
- `Done`
- `Superseded`
